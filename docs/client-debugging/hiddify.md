# Hiddify

Launch path:

- Runner: `ProxyAppClientRunner`
- Harness entry: `createClientRunner({ client: "hiddify" })`
- Runtime config: Hiddify JSON wrapper generated by `wrongsv
  --print-client-config --format hiddify`, plus manual builders for protocols
  not yet emitted natively.

Debug surface:

- Primary debug adapter: `VmBridgeDebugClient`
- Source: Flutter VM service exposed by `ProxyAppManager`
- Runtime tweaks: extension inventory only; no stable live protocol mutation
  API from the GUI shell.

Testing integration:

- Headless GUI launch through `ProxyAppManager`
- Proxy port is provided by the running app and waited on before traffic starts
- Hiddify XHTTP uses the custom embedded `type: "xray"` outbound wrapper from
  the packaged core

Artifacts:

- `debug-initial.json`
- `debug-tweaks.json`
- `debug-final.json`
- `report.json`

Client-side limitations:

- `anytls_tcp`: packaged core on this box rejects `type: "anytls"` as unknown.
  Treat as client/runtime gap, not `wrongsv`.
- Other currently covered families include `shadowtls_tcp`, `hysteria2_tcp`,
  `tuic_tcp`, and `vless_xhttp`.
